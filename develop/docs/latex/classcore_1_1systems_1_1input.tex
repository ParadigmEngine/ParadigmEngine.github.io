\doxysubsection{Detailed Description}
the input class is responsible for translating platform specific input events to a more generalised format, and sending out events that have been subscribed to. for example it will translate all keyboard input messages into a generic key message that remains the same for all platforms. You can however reverse-\/translate back into the platform specific key messages. Every input instance is tied to a \mbox{\hyperlink{classcore_1_1os_1_1surface}{core\+::os\+::surface}}, and all its data (such as mouse coordinates) is invariably tied to that regardless if the actual platform only has a singleton input system. \begin{DoxyWarning}{Warning}
all input state persists till the next platform specific message loop. this means that the \mbox{\hyperlink{structcore_1_1systems_1_1input_1_1mouse__delta}{mouse\+\_\+delta}} can be the same for several frames untill the input system gets an update. that\textquotesingle{}s why you should prefer listening to the events rather than iterating the input system itself. 
\end{DoxyWarning}
